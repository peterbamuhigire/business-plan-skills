# Phase 06 - Proposal and Grant Engine Expansion

## Objective

Turn the repository into a complete proposal-production engine, not just a business-plan engine, by strengthening project proposals, business proposals, grant proposals, and related packaging workflows.

## Why This Phase Matters

Your stated ambition includes:

- business plans
- project proposals
- business proposals
- grant proposals

The repo already contains strong raw materials, especially `proposal-architect` and `11b-grant-proposal`, but they need tighter integration with the rest of the advisory system.

## Core Workstreams

### 1. Strengthen `proposal-architect`

Refine it so it works as the standard orchestration layer for client-facing proposals. Add:

- clearer qualification logic
- value proposition design by buyer type
- win-theme development
- evidence and case-study packaging
- pricing logic by proposal type
- final review gates aligned to evaluation criteria

### 2. Build proposal-type variants

Create explicit workflows for:

- consulting proposals
- implementation proposals
- strategic advisory proposals
- donor-funded project proposals
- partnership and business-development proposals

### 3. Upgrade grant capability

Enhance `11b-grant-proposal` and related meta-skills to include:

- stronger Theory of Change depth
- donor-language adaptation
- budget narrative discipline
- M&E integration
- safeguarding, sustainability, and compliance expectations

### 4. Link proposals to the business-plan core

Make sure proposal outputs can reuse:

- country context
- evidence matrix
- financial logic
- consulting synthesis
- due-diligence thinking

This avoids maintaining two separate quality systems.

## Repository Changes to Make

- upgrade `proposal-architect/SKILL.md`
- enhance `11b-grant-proposal`
- create reference files for win themes, proposal economics, and donor logic
- add routing notes from proposal work into relevant meta-skills

## Deliverables

- upgraded proposal orchestration
- proposal-type workflow variants
- stronger grant proposal system
- tighter link between proposal work and core plan logic

## Exit Criteria

Phase 06 is complete only when:

- the system can produce both opportunity-specific proposals and reusable proposal packages
- grant proposals are analytically strong, not only compliant
- proposal pricing, delivery, and staffing logic align reliably
- proposal outputs feel like a natural extension of the same advisory engine

## Risks if Skipped

- the repo remains strongest only in business-plan work
- proposals feel disconnected from the deeper strategy and finance logic
- grant outputs stay compliant but not competitively differentiated

## Suggested Reading Material

- *The Business of Consulting* by Elaine Biech
- *Flawless Consulting* by Peter Block
- *HBR Guide to Winning Business Plans* by Edward Blackwell
- *Proposal Writing* by Lynn E. Miner and Jeremy T. Miner

Buy and provide these here because this phase depends on winning work commercially, structuring proposals persuasively, and translating expertise into client-buyable packages.
