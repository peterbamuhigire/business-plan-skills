# Phase 08 - Productisation, Delivery Assets, and Team Enablement

## Objective

Convert the repository from founder-held expertise into a repeatable consultancy production system that can be delivered by a team with consistent quality.

## Why This Phase Matters

A true advisory engine is not only about content quality. It must also be:

- teachable
- repeatable
- commercially packageable
- operationally scalable

Without this phase, the system may produce excellent work but remain too dependent on one operator.

## Core Workstreams

### 1. Build standard engagement packs

Create reusable client-facing and internal assets such as:

- intake questionnaire packs
- evidence request lists
- interview guides
- proposal shells
- report skeletons
- appendix checklists
- presentation shells
- handover packs

### 2. Create consultant playbooks

Document how a junior or mid-level consultant should use the system for:

- a bankable plan
- an investor plan
- a grant proposal
- a business proposal
- a strategic review engagement

### 3. Package commercial offers

Define productised service packages with:

- scope
- turnaround time
- inputs required from client
- exclusions
- review rounds
- pricing logic

### 4. Establish training and calibration

Introduce:

- onboarding curriculum
- sample engagement walkthroughs
- scoring rubrics
- peer-review routines
- periodic quality calibration sessions

## Repository Changes to Make

- add delivery templates under `docs/` or a dedicated assets area
- write consultant playbooks
- add package definitions and workflow examples
- create example engagement artefacts where safe

## Deliverables

- reusable engagement assets
- consultant operating playbooks
- productised service packages
- onboarding and calibration materials

## Exit Criteria

Phase 08 is complete only when:

- a trained consultant can run the system without founder-level improvisation
- clients can buy clear packaged offers
- delivery quality is teachable and auditable
- the engine is closer to a firm capability than a personal knowledge base

## Risks if Skipped

- high-quality work but weak scalability
- poor delegation and onboarding
- difficulty commercialising the engine consistently
- revenue limited by founder bandwidth

## Suggested Reading Material

- *Managing the Professional Service Firm* by David Maister
- *The E-Myth Revisited* by Michael Gerber
- *The Trusted Advisor* by David Maister, Charles Green, and Robert Galford
- *The Consulting Bible* by Alan Weiss

Buy and provide these here because this phase is about turning expertise into a repeatable service operation with standards, packaging, and team usability.
