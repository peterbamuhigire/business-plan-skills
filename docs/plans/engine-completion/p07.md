# Phase 07 - Consulting Synthesis and Quality Assurance

## Objective

Institutionalise top-tier consulting discipline so that every important deliverable has a governing thesis, a decision-ready storyline, a contradiction review, and a final quality gate before release.

## Why This Phase Is the Reputation Layer

This is the phase that turns strong content into premium advisory work. The repo already has `meta-consulting-synthesis`, `meta-bankability-scoring`, `meta-due-diligence`, and `meta-financial-stress-test`. The next step is to make them operate as a coherent review stack.

## Core Workstreams

### 1. Make consulting synthesis mandatory

For high-stakes outputs, require `meta-consulting-synthesis` before final assembly. It should become the formal bridge between draft content and client-ready narrative.

### 2. Create a unified review stack

Define the review order for major engagements:

1. consulting synthesis
2. market validation
3. financial stress test
4. bankability or valuation review
5. due-diligence review
6. appendix and evidence audit

### 3. Build release gates by audience

Create go/no-go checklists for:

- bank submission
- investor circulation
- DFI discussion
- donor submission
- client proposal submission
- board strategy presentation

### 4. Introduce benchmark reviews

Build a standard internal red-team exercise:

- strongest objection
- missing evidence
- hidden assumption
- weak slide or section
- likely diligence question
- likely committee objection

## Repository Changes to Make

- add a unified review-order reference
- strengthen cross-links among all meta-skills
- create audience-specific quality-gate templates
- create a red-team review template under `docs/`

## Deliverables

- unified review stack
- audience release gates
- red-team review format
- stronger mandatory role for consulting synthesis

## Exit Criteria

Phase 07 is complete only when:

- every serious deliverable passes through an explicit review sequence
- release decisions are based on documented gates
- contradictions and weak assumptions are systematically surfaced before delivery
- the system behaves more like an advisory firm and less like a writing toolkit

## Risks if Skipped

- strong drafts still leave avoidable weaknesses unchallenged
- quality remains personality-dependent instead of system-driven
- premium positioning is undermined by inconsistent final polish

Buy and provide these here because this phase is about sharpening the logic, challenge process, and executive communication standard of final outputs.
