# Phase 03 - Research, Evidence, and Country Intelligence

## Objective

Raise the evidence quality of the entire system by making primary research, source hierarchy, claim labelling, and country-context depth mandatory where stakes are high.

## Why This Phase Matters

Large firms win trust partly because their documents signal disciplined evidence handling. This repo already has strong frameworks, but the next leap requires stronger proof architecture:

- better primary research protocols
- better country-specific context
- stronger evidence classification
- explicit data-confidence rules

## Core Workstreams

### 1. Create a dedicated primary-research meta-skill

Build the missing `meta-primary-research` skill recommended in the repo evaluation. It should cover:

- interview guides
- customer discovery protocols
- survey standards
- mystery shopping
- supplier quotation capture
- observation protocols
- triangulation methods
- claim-confidence grading
- evidence packaging for appendices

### 2. Upgrade market-facing skills

Strengthen:

- `04-market-analysis`
- `05-target-market`
- `06-competitive-analysis`
- `07-marketing-sales-strategy`
- `meta-market-validation`

Each should require:

- an explicit hypothesis
- evidence hierarchy labels
- confidence flags
- stronger customer-proof logic
- stronger competitor-economics discipline

### 3. Expand country context beyond Uganda

Prioritise additional country files, beginning with:

- Rwanda
- Kenya
- Tanzania

Then create a maturity audit for every country file covering:

- tax basics
- labour context
- registration and licensing
- financing environment
- sector-specific regulatory issues
- FX and inflation risk notes

### 4. Build an evidence matrix standard

For all serious outputs, create a simple matrix that links each major claim to:

- source type
- source date
- confidence level
- appendix evidence
- unresolved proof burden

## Repository Changes to Make

- Add `meta-primary-research/`
- add or upgrade reference files for market and customer proof
- expand `country-context/`
- create a reusable evidence-matrix template in `docs/`

## Deliverables

- new `meta-primary-research` skill
- upgraded market-validation workflow
- stronger market/customer/competition skills
- expanded country-context coverage
- evidence matrix template

## Exit Criteria

Phase 03 is complete only when:

- a reviewer can distinguish assumptions from evidence instantly
- market-facing claims show source quality and confidence level
- at least the top target countries have credible baseline context
- appendices can support major claims without last-minute scrambling

## Risks if Skipped

- polished plans with weak factual footing
- low trust from funders, investors, and sophisticated clients
- repeated market claims that cannot survive diligence
- inability to scale quality because evidence logic remains informal

## Suggested Reading Material

- *The Mom Test* by Rob Fitzpatrick
- *Testing Business Ideas* by David Bland and Alexander Osterwalder
- *HBR Guide to Building Your Business Case* by Raymond Sheen and Amy Gallo
- *Competitive Strategy* by Michael Porter

Buy and provide these here because this phase is about how to gather, classify, and defend evidence rather than merely write around it.
