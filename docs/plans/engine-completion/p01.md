# Phase 01 - Strategic Positioning and Target Operating Model

## Objective

Define exactly what the consultancy is building, who it serves best, what output tiers it will own, and where it will aim to beat larger firms. Without this phase, the repository risks becoming broad but commercially unfocused.

## Why This Phase Comes First

The repo already contains many useful skills. The bigger risk is strategic sprawl. A world-class system must know:

- its priority client types
- its priority output types
- its proof standard by audience
- its positioning relative to banks, investors, DFIs, grant makers, and corporate clients

## Core Workstreams

### 1. Define the consultancy thesis

Write a one-page internal strategy statement covering:

- the firm's governing thesis
- priority geographies
- priority sectors
- priority buyer types
- why clients should choose this engine over a traditional consultant

### 2. Set service-line architecture

Formalise five service lines and their output expectations:

1. business plans
2. project proposals
3. business proposals
4. grant proposals
5. strategy and execution programmes

For each line, define:

- ideal client
- decision-maker
- buying trigger
- required sections
- review gates
- commercial packaging

### 3. Define maturity tiers

Create explicit standards for:

- SME bank-loan plan
- investor/equity plan
- DFI-impact plan
- grant proposal
- strategic-partner proposal
- board-ready strategy pack

This should build directly on the recommendation already identified in `docs/evaluation/2026-04-12/recommendations.md`.

### 4. Clarify where you can beat major firms

Write a positioning note that states:

- where you can credibly outperform large firms
- where you will not try to compete head-on
- what "world-class" means in East African and frontier-market contexts

## Repository Changes to Make

- Add a strategy document under `docs/` that defines service lines, target clients, and quality tiers.
- Update `README.md` so the suite is described as a consultancy engine, not only a skill collection.
- Add a short "audience mode" reference table that all major skills can reuse.

## Deliverables

- Consultancy thesis document
- Service-line architecture document
- Audience-mode quality tiers
- Positioning note against Big Four / strategy boutiques
- Updated repository narrative in core docs

## Exit Criteria

Phase 01 is complete only when:

- the consultancy can describe its offer in one page without ambiguity
- each major output type has a named standard
- "world-class" is operationally defined, not aspirationally described
- the rest of the roadmap can refer back to one agreed operating model

## Risks if Skipped

- too many overlapping skills without a clear commercial path
- inconsistent quality standards by document type
- effort spent upgrading low-value areas while strategic bottlenecks remain
- weak differentiation against generic consulting and writing services

## Suggested Reading Material

- *Good Strategy Bad Strategy* by Richard Rumelt
- *Playing to Win* by A.G. Lafley and Roger Martin
- *The Pyramid Principle* by Barbara Minto
- *The McKinsey Mind* by Ethan Rasiel and Paul Friga

Buy and provide these first because they will shape the logic, positioning, and structuring standard for every later phase.
