**Score: 8.5 / 10**

## 1.3 Framework Orchestration Gap

**The most significant issue is not missing frameworks — it is that existing frameworks are available as references but not enforced as checkpoints.**

McKinsey/Bain enforce discipline through process: every engagement starts with an issue tree, every recommendation has a so-what test, every financial projection has explicit sensitivity disclosure.

Current orchestration gaps:
- `meta-market-validation` contains McKinsey MECE/issue tree methodology but the core skills (04, 05, 06) do not *require* an issue tree before analysis begins
- `meta-bankability-scoring` CAMPARI checklist exists but is not invoked automatically after `10-financial-projections` or `11-funding-request`
- `meta-financial-stress-test` exists but there is no explicit trigger requiring it before plan submission
- `00-plan-assembly` addresses funder type and covering letters but does not contain a mandatory pre-flight checklist that enforces all meta-skills sequentially

**Recommendation:** Create a mandatory Quality Gate Checklist in `00-plan-assembly` that references all validation meta-skills with explicit go/no-go criteria before finalising any section.
