# Logical Framework (LogFrame) Templates

*The LogFrame (Logical Framework Matrix) is the standard planning and accountability tool for development partner grants. It is required by the EU, UN agencies, GIZ, USAID (as Result Framework), DFID/FCDO, and most bilateral donors. Master this before submitting any grant proposal.*

---

## 1. What a LogFrame Is and Why It Matters

A LogFrame answers four questions in a single matrix:
1. **What will the project achieve?** (Objectives and results hierarchy)
2. **How will we know if it worked?** (Objectively Verifiable Indicators — OVIs)
3. **Where will we get evidence?** (Means of Verification — MoV)
4. **What could stop it working?** (Assumptions and risks)

The hierarchy runs from bottom to top:
```
Impact (Goal)          ← Long-term change in society (5–10 years)
    ↑
Outcome (Purpose)      ← Change in behaviour or conditions (2–4 years)
    ↑
Outputs                ← Tangible deliverables produced by activities
    ↑
Activities             ← What the project actually does
    ↑
Inputs                 ← Resources: staff, money, materials
```

## 4. Donor-Specific LogFrame Variations

| Donor | Tool Name | Key Differences |
|---|---|---|
| **EU** | LogFrame Matrix (LFM) | 5 columns (adds "Budget" column per output); uses "Overall Objective" / "Project Purpose" language |
| **GIZ** | LogFrame (standard) | DAC criteria (Relevance/Coherence/Effectiveness/Efficiency/Impact/Sustainability) used in evaluation |
| **USAID** | Results Framework (RF) | No LogFrame; hierarchical Results Framework with IR (Intermediate Results) and Sub-IRs; narrative-based |
| **DFID/FCDO** | Theory of Change + Results | ToC narrative required; Results Framework with logframe-style indicators |
| **UN agencies (UNDP, UNICEF, WFP)** | UNDAF-aligned LogFrame | Must link to UNSDCF (UN Sustainable Development Cooperation Framework) country priorities |
| **World Bank** | Results Framework and M&E Plan | PDO Indicators (Project Development Objective); intermediate outcome indicators; disbursement-linked indicators (DLIs) for some programmes |
| **Uganda Government (YLP, PDM, Emyooga)** | Work Plan Matrix | Simplified: Activities → Outputs → Targets → Timeline → Responsible Person; no formal LogFrame required |

## 5. Common LogFrame Mistakes

1. **Confusing outputs with outcomes** — "500 farmers trained" is an output (you can count it immediately). "Farmers adopt improved practices" is an outcome (requires behaviour change, takes time).

2. **Unmeasurable indicators** — "Improve farmer livelihoods" cannot be measured. Replace with "Average monthly household income increases from UGX 450,000 to UGX 620,000 by Year 3."

3. **Missing baselines** — An indicator without a baseline is useless. If you cannot state the starting point, you cannot prove change.

4. **Weak assumptions column** — Assumptions are real risks, not formalities. "Government remains supportive" is too vague. "UCDA maintains its EUDR traceability database without service disruption" is specific and testable.

5. **Activities that don't logically produce the outputs** — Check the vertical logic: do your activities actually produce your outputs? Do your outputs actually cause your outcome?

6. **Gender-blind indicators** — Most donors require gender-disaggregated data. "500 farmers trained" should be "500 farmers trained (300 women / 200 men)." Add a gender column or disaggregate all people-related OVIs.
