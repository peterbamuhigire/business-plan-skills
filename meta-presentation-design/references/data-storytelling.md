# Data Storytelling Standards

Use this file when designing charts, tables, and evidence-led slides.

## 1. Start with context

Before choosing a chart, define:

- who the audience is
- what decision they need to make
- what one takeaway they must remember

If the takeaway is unclear, the chart choice will also be weak.

## 2. Choose the simplest valid display

Prefer:

- bars for comparison
- lines for time trends
- waterfalls for bridge logic
- scatter plots for relationship questions

Avoid chart types that require decoding effort for routine business decisions.

## 3. Eliminate clutter

Remove:

- unnecessary borders
- background shading
- repeated legends when direct labels work
- excessive decimals
- decorative icons that do not add meaning

Every non-data element must earn its place.

## 4. Focus attention intentionally

Use contrast, annotation, and placement to direct the eye:

- highlight the key bar or line
- mute secondary series
- label the number that matters most
- add a short action title stating the conclusion

## 5. Move from dashboard to story

A dashboard shows many metrics.
A decision slide should answer one question.

When using dashboard outputs in a deck:

1. isolate the one metric that matters now
2. explain what changed
3. explain why it matters
4. state the decision implication

## 6. Executive chart rules

- use action titles, not neutral titles
- annotate turning points
- keep tables for backup or appendix unless the audience must inspect exact values
- never force the audience to calculate the conclusion themselves
